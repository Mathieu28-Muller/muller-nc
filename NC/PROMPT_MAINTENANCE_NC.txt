================================================================================
RÈGLE CRITIQUE — "EN RETARD RÉEL" (ISO §10.2.1 — ajoutée 29/05/2026)
================================================================================

  Une action CAPA est "en retard" UNIQUEMENT si :
    1. echeance < aujourd'hui
    2. ET datePiloteReponse est null OU datePiloteReponse > echeance

  Si datePiloteReponse <= echeance → le pilote a répondu dans les délais.
  L'action est "En attente de validation admin" — non imputable au pilote.
  NE PAS compter cette action dans les retards (dashboard, stats, fiche, timeline).

  Champ JSON : datePiloteReponse (camelCase)
  Champ PG   : date_pilote_reponse (snake_case)
